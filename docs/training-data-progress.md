## Setup Required

### Environment Variables Needed

Add these to `.env.local`:

```
# Laminar — LLM trace capture (get from https://laminar.sh dashboard)
LMNR_PROJECT_API_KEY=your_laminar_project_api_key

# Convex — game data persistence (auto-generated by npx convex dev)
NEXT_PUBLIC_CONVEX_URL=https://your-project.convex.cloud
```

### Convex Project Setup

Convex requires a one-time interactive setup:

```bash
npx convex dev
```

This will:
1. Prompt you to log in (GitHub auth)
2. Create a Convex project
3. Generate proper `convex/_generated/` files (replacing the placeholders)
4. Set `NEXT_PUBLIC_CONVEX_URL` in `.env.local`
5. Deploy your schema and functions

After setup, `npx convex dev` also runs in watch mode to sync changes.

### Laminar Project Setup

1. Go to https://laminar.sh and sign up
2. Create a project
3. Copy the project API key
4. Add it to `.env.local` as `LMNR_PROJECT_API_KEY`

### Graceful Degradation

Both integrations are **optional** — if env vars aren't set:
- Laminar: prints a warning, tracing disabled, game works normally
- Convex: prints a warning, data not persisted, game works normally

The game itself is never blocked by missing data collection config.
